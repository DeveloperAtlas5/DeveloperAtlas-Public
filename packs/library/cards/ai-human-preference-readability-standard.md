# AI Human Preference & Readability Standard

**Level:** AI Collaboration

## What it means

The AI Human Preference & Readability Standard is a rule set for AI-generated code.

It tells AI to prioritize human-readable, maintainable code over clever or overly fast output.

## Why it exists

AI can write working code that is hard for a learner to understand.

That creates a problem:

```txt
the code works
but the human loses control
```

Developer Atlas does not want that.

## Use it when

Use this standard when you ask AI to:

- explain code
- review code
- change code
- fix an error
- generate a small patch
- keep a project readable

## Core rule

```txt
Clarity over cleverness.
Human control over AI autonomy.
Learning value over speed.
```

## Tiny prompt

```txt
Follow the AI Human Preference & Readability Standard.

Prioritize readable, maintainable code for a rookie-to-junior developer.
Explain first.
Suggest a small approach.
Patch only the smallest necessary change.
Park unrelated ideas.
```

## Watch out

Do not use this standard as a vague style wish.

Use it as an active review checklist.
